MVP Task Breakdown (User Stories)

We will break the project into 3 "Sprints" (1 week each).
Sprint 1: The "Search & Fetch" Core

Focus: Getting data from APIs and basic scraping.
ID	Task Name	Est. Time	Definition of Done (DoD)
1.1	Project Skeleton	4 Hours	Git repo set up, poetry/pip env created, folder structure ready.
1.2	HF Integration	6 Hours	Script can print top 5 datasets from Hugging Face for a query.
1.3	Kaggle Integration	6 Hours	Script can authenticate and list datasets from Kaggle.
1.4	Basic Scraper	10 Hours	Crawlee script that can go to a Wikipedia page and download a table into CSV.
Sprint 2: The "Compliance & Logic" Brain

Focus: Making the data usable and legal.
ID	Task Name	Est. Time	Definition of Done (DoD)
2.1	License Regex	5 Hours	Function detects "MIT" or "CC-BY" in a text string.
2.2	Splitter Logic	8 Hours	Function takes a CSV and saves train.csv and test.csv (80/20 split).
2.3	Formatter	6 Hours	Function converts the CSV into .jsonl (Chat format for LLMs).
2.4	Pipeline Glue	8 Hours	A script that connects Sprint 1 & 2: Search -> (if fail) Scrape -> Split -> Save.
Sprint 3: The "User Interface"

Focus: Building the Streamlit frontend.
ID	Task Name	Est. Time	Definition of Done (DoD)
3.1	Search UI	5 Hours	Streamlit search bar that displays API results in a table.
3.2	Scrape UI	6 Hours	Input box for URL + Button "Scrape & Process".
3.3	Download UI	4 Hours	Buttons to download the final .zip file.
3.4	MVP Deploy	4 Hours	App running locally without crashing on errors.
Part 3: The Timeline

Assuming you work on this part-time (e.g., 2-3 hours a day), here is a realistic schedule:
Week	Phase	Key Milestone
Week 1	Foundation	You can type "Cancer Data" and get a list of results from HF/Kaggle in your terminal.
Week 2	The Scraper	You can paste a Wikipedia URL, and it saves a CSV to your laptop automatically.
Week 3	Processing	The system automatically splits that CSV into Train/Test JSONL files.
Week 4	Interface	You have a web UI (Streamlit) where you can click buttons instead of running scripts.









The Roadmap:
Phase 1 (Week 1): Layer 1 - Federated API Search (HF + Kaggle)
Phase 2 (Week 1-2): Layer 1.5 - Search Bridge + URL Filter
Phase 3 (Week 2): Layer 2 - Scraper + Circuit Breaker
Phase 4 (Week 3): Layer 3 - Compliance + AI Cleaning
Phase 5 (Week 3): Layer 4 - Splitting + Export
Phase 6 (Week 4): FastAPI Orchestration
